# YOLO Model Factory 操作教程

本文说明本地平台的完整闭环。Web 地址为 http://127.0.0.1:53257/，API 文档为 http://127.0.0.1:8000/docs。

## 1. 创建任务合同

先确定任务类型 `detect` 或 `segment`，再确定 class 名称和顺序。class 顺序会成为数据集、训练和模型的稳定合同；正式发布数据集后，不要在同一版本中删除、重排或改变含义。

仓库只保留匿名化任务配置示例；真实业务任务和类别定义不得提交到 Git。

## 2. 导入数据

有两种入口：

- 图片：在数据集/图片导入入口一次选择多张图片。系统会建立图片导入 collection 和 frame batch，图片以 `selected` 状态进入复核与标注队列。
- 视频：在视频页面从浏览器选择一个或多个视频上传，再创建抽帧批次，设置抽帧间隔和图片质量。候选帧经过近重复检查后进入人工筛选。

浏览器上传不会修改用户电脑上的原文件。云端服务不能直接读取用户电脑的绝对路径；文件上传后按 SHA-256 归档，并保留来源追踪信息。

## 3. 筛选和标注

在数据复核页查看批次图片，将候选帧设置为保留或拒绝。点击标注入口进入原生编辑器：

- detect：绘制、移动和删除矩形框。
- segment：绘制多边形、拖动顶点和删除形状。
- 每个形状都可选择本任务合同中的 class。
- 修改立即按修订号持久化；并发旧修订会被拒绝。
- 复核通过后编辑器锁定；需要修改时先退回待修正状态。
- segment 可选择 `sam2_t.pt` 或 `sam2_s.pt`，在目标中心点击生成建议多边形，再人工修整。

SAM2 第一次运行需要下载权重。若 GitHub 网络不稳定，可预先把官方权重放在项目工作目录中，文件名保持 `sam2_t.pt` / `sam2_s.pt`。

## 4. 发布数据集

复核完成后导出原生 YOLO 标注。发布前系统检查：

- 图片可解码，图片与标签一一配对。
- class ID 在任务合同范围内。
- detect 标签字段和归一化框合法。
- segment 多边形点数和归一化坐标合法。
- 同一源视频的帧不会跨 train/val/test 泄漏。

发布结果是不可变的 `dataset-vX.Y.Z`，包含 `data.yaml`、train/val/test、manifest、校验报告、类别统计和校验和。后续纠错或新增数据必须发布新版本。

使用 Roboflow 外部标注时，在“回导标注 ZIP”中直接选择 Roboflow 导出的 `.zip`。系统会先上传到受管临时目录，完成安全解压、类别和格式校验后归档，并自动清理临时 ZIP；不要填写本机或服务器路径。

数据集详情页可浏览该版本内的全部图片。上传新图片不会修改已发布版本，而是进入新的图片导入批次。

## 5. 训练模型

在训练页面选择：

- 已发布数据集。
- detect 或 segment 对应基础权重。
- YOLOv8、YOLO11、YOLO26 官方预设，或通过页面上传一个自定义 `.pt` 权重。
- 本次参与训练的 class 子集和可选显示别名。
- epochs、imgsz、batch 和 device。

训练在独立进程中执行，页面显示实时进度和日志。可以取消运行；API 重启后会根据 PID、manifest 和进度文件恢复或标记中断状态。

训练创建、安全重试和独立评估开始前，系统会先删除可再生成的缩略图缓存、超过保留期的上传暂存目录和临时文件，然后重新检查磁盘。默认要求同时保留至少 `8 GiB` 和 `10%` 空闲空间；在小容量云盘上仍建议实际保持 `10-12 GiB` 以上。自动清理不会触碰 SQLite、数据集、原始图片、标注、模型权重或正式训练产物。

当训练收到 `exit -9` 或 `137` 时，`resource_limit` 只表示进程遭到外部 `SIGKILL`。系统会比较训练开始前后的 cgroup `oom_kill` 计数，并记录内存限制、当前值和峰值；只有本次运行期间计数增加才确认 OOM。失败详情中的“安全重试”会创建参数更保守的新任务，不修改原失败记录；请求 ID 在 HTTPS 和普通 HTTP 部署中均可生成。

## 6. 注册和发布模型

训练完成且存在 `best.pt` 后，从训练详情注册候选模型。然后在模型中心运行门禁：

1. 校验训练状态和 PT 制品。
2. 导出 ONNX opset 17。
3. 用验证样本分别执行 PT 和 ONNX。
4. 比较类别、置信度、框以及分割 mask 的一致性。
5. 保存门禁报告、环境信息和制品 SHA-256。

四项门禁 `training`、`pt`、`onnx`、`consistency` 全部通过后才能发布。已发布模型必须先归档才能删除；被推理历史引用的模型不能删除。

## 7. 异步推理

在推理工作台选择已发布模型、运行时和输入模式：

- `PT / CUDA`：使用 GPU。
- `ONNX / CPU`：便于无 GPU 部署和一致性验收。
- 单图模式上传一张图片，批量模式可一次选择多张图片，视频模式上传一个视频文件。

创建任务后 API 立即返回，独立 Runner 在后台执行。页面轮询任务状态，可取消运行中的任务，并可在历史记录中重新打开结果。产物位于：

```text
<storage-root>/inference-runs/<run-id>/
|-- manifest.json
|-- runner.log
|-- progress.jsonl
|-- result.json
`-- outputs\annotated\
```

服务重启后，已完成任务及其结果仍可查看；仍存活的任务恢复跟踪，失去子进程的任务会被标记为中断/失败，而不会永久停在运行中。

## 8. 删除规则

各资源页面均提供删除入口，但系统保护依赖关系：

- 运行中的训练或推理不能删除。
- 已被模型引用的训练不能删除。
- 已发布模型必须先归档。
- 被推理历史引用的模型不能删除。
- 被训练或模型引用的数据集不能删除。
- 含抽帧批次的视频 collection 不能直接删除。

“删除记录”默认保留磁盘产物；“删除记录并删除制品”只允许删除存储根目录内经过路径校验的文件。推理上传的源文件也会随“删除记录并删除制品”清理。

## 9. 日常启动和验收

API：

```powershell
Set-Location "<TrainForge仓库目录>"
$env:PYTHONPATH = "src"
python -m uvicorn yolo_factory.api.app:create_app --factory --host 127.0.0.1 --port 8000
```

Web：

```powershell
Set-Location "<TrainForge仓库目录>\frontend"
npm run dev -- --host 127.0.0.1 --port 53257 --strictPort
```

全量验收：

```powershell
$env:PYTHONPATH = "src"
python -m pytest

Set-Location frontend
npm test -- --run
npm run build
```

## 10. 云端迁移口

当前资源 API、Repository、Executor、Runner 和不可变 manifest 已分层。云端阶段可保持前端和主要 API 合同，替换数据库、任务调度和文件存储，并增加认证、租户、配额与审计，不需要重写训练和推理业务流程。
# 类别标识与显示名

新建任务时，每个类别包含两个名称：

- 英文标识（例如 `scratch`）是稳定的训练名称，与 class id 绑定。
- 中文显示名（例如“划痕”）用于任务、标注和训练页面识别。

界面会显示为“划痕（scratch）”，但 YOLO 标签、导出和训练仍使用 `scratch` 及原 class id。标注时选中已有对象，可在右侧“已选对象类别”中更改类别。

已有任务需要补充或修改中文名称时，进入“任务管理”，点击操作列中的铅笔按钮，在每个锁定的 Class ID 和英文标识旁填写“中文显示名称”并保存。系统不允许修改已有任务的 Class ID、英文标识或类别顺序，因为这些字段已经与历史 YOLO 标签绑定。任务的更多菜单提供安全删除和级联删除：安全删除在存在下游数据时会拒绝；级联删除会永久清理该任务的图片、视频、标注、数据集、训练、模型、推理记录及托管产物。

# SAM2 首次加载

API 进程对每个 SAM2 模型只加载一次。首次选择 Tiny 或 Small 可能需要 15–30 秒；后续增加正负点会复用已加载模型。重启 API 或首次切换到另一个模型时会再执行一次加载。
