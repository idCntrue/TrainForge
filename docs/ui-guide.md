# TrainForge 界面指南

TrainForge Web 界面覆盖从数据进入到模型推理的单机闭环。默认本地开发地址为 <http://127.0.0.1:53257>，Docker 地址由 `WEB_PORT` 决定。

## 工作台

工作台展示任务、数据批次、标注、数据集版本、训练、模型和推理的汇总状态。统计来自 API 和 SQLite 注册表，不扫描用户电脑上的任意目录。

## 数据准备

- **任务管理：** 创建 detect/segment 任务，维护稳定的 class 标识和可修改的显示名称。
- **数据导入：** 从浏览器上传图片或视频；云端页面不会读取本机绝对路径。
- **数据筛选：** 分页查看候选帧、跨页批量修改状态、追加图片或视频，并通过七天回收站恢复或永久删除。
- **原生标注：** 编辑检测框或分割多边形，使用修订号避免并发覆盖，并在复核通过后锁定结果。
- **数据集版本：** 发布经过结构和质量校验的不可变 train/val/test 数据集。

## 模型开发

- **训练运行：** 选择数据集、基础权重、设备和资源预设，查看 Epoch、指标、日志、失败诊断和恢复入口。
- **模型中心：** 将训练结果登记为候选模型，运行 PT/ONNX 一致性门禁，再发布或归档。
- **推理工作台：** 使用已发布模型处理单图、批量图片或视频，并保留异步任务历史。

## 训练资源提示

训练开始前会先清理可再生成缓存和过期暂存文件，再执行 `8 GiB` 与 `10%` 双重磁盘检查。自动清理不会删除数据库、数据集、标注或权重。

`resource_limit` 表示训练进程被外部强制终止。只有 cgroup OOM 计数在本次运行期间增加，页面才应将其解释为已确认内存不足；否则应查看技术诊断和服务器日志。安全重试会创建参数更保守的新运行，不修改原失败记录。

## 本地启动

在仓库根目录安装依赖后，可以使用：

```powershell
./scripts/start-ui.ps1
```

或分别启动 API 与前端：

```bash
uvicorn yolo_factory.api.app:create_app --factory --host 127.0.0.1 --port 8000

cd frontend
npm run dev -- --host 127.0.0.1 --port 53257 --strictPort
```

生产环境推荐通过 Docker Compose 的同源 Nginx 入口访问，不直接暴露 API 容器端口。
