# 训练实时概览与运行日志工作台设计

## 背景与目标

当前训练详情将进度、单个主指标、单个 Loss、数据集划分和全部配置平铺在“实时概览”中，关键信息缺少层级；运行日志位于“参数与产物”末尾，查找故障和跟踪训练过程都不方便。

本次仅调整前端展示，不修改训练请求、训练执行、轮询频率、后端接口或持久化数据。Vite 热更新应使正在运行的训练不受影响。

目标：

- 用户打开详情后，先回答“训练到哪了、还要多久、当前效果如何、Loss 是否在改善”。
- 将不同 Loss 和验证指标明确分开，避免把某一个 Loss 当作模型质量结论。
- 让数据划分和关键配置可快速扫描，类别名称不再挤成一行。
- 将日志提升为独立页签，支持搜索、过滤、定位和复制。
- 对暂无数据的状态说明生成条件，不再只显示“待生成”。

## 范围

### 包含

- 重构“实时概览”页签。
- 新增独立“运行日志”页签。
- 从“参数与产物”移除内嵌日志区，保留完整日志文件作为可下载产物。
- 新增纯前端的数据展示辅助函数及组件测试。
- 完成桌面端和移动端响应式适配。

### 不包含

- 不新增或修改后端 API。
- 不改变训练参数、训练进程、资源门禁、日志写入或轮询机制。
- 不提供行业通用“正常值区间”；不同任务和数据集不可用统一阈值判断。
- 不伪造背景图数量、验证轮次或缺失的指标。
- 不读取浏览器本地文件路径；完整日志仍通过已有产物下载接口获取。

## 信息架构

训练详情保留以下顶层页签：

1. 实时概览
2. 指标曲线
3. 结果图像
4. 运行日志
5. 参数与产物

“实时概览”按决策优先级排列：

1. 运行状态条
2. 核心质量指标
3. Loss 分解
4. 数据集划分
5. 关键配置
6. 现有质量报告、失败诊断和警告

失败诊断、完成质量报告和数据质量证据继续保留，但不重复制造第二套判断逻辑。

## 实时概览设计

### 运行状态条

采用紧凑的横向状态区，显示：

- 当前阶段中文名称。
- 当前轮次 / 总轮次。
- 完成百分比。
- 已运行时间：使用现有运行记录的 duration 展示。
- 单轮平均耗时：使用 `details.timing.epoch_seconds`。
- 预计剩余时间：使用 `details.timing.eta_seconds`。

移动端改为两列或单列排列，不使用占据大量高度的圆形进度图。状态结束后，预计剩余时间显示“已完成”或“已结束”。

### 核心质量指标

检测任务显示：

- Box mAP50
- Box mAP50-95

分割任务显示：

- Mask mAP50
- Mask mAP50-95
- Box mAP50
- Box mAP50-95

数值来源仅限 `details.latest_metrics` 或 `epoch_history` 最新一条真实记录。缺失时显示“首轮验证完成后生成”；没有历史数据时不回退为 0。

每个指标附简短解释：mAP50 更容易理解当前检出能力，mAP50-95 对定位或轮廓精度要求更严格。说明“通常越高越好”，但不提供跨数据集固定合格线。

### Loss 分解

根据逐轮数据实际存在的键显示：

- Box Loss：目标位置偏差。
- Mask Loss / Seg Loss：分割轮廓偏差，仅分割任务。
- Class Loss：类别判断偏差。
- DFL Loss：边界框细粒度定位偏差。

每项显示最新值和最近最多 5 轮趋势。趋势仅描述“下降、变化较小、上升、数据不足”，不直接宣称模型好坏。Loss 数值不可跨任务横向比较。

### 数据集划分

显示训练集、验证集、测试集的图片数量、百分比和总数。每一类附一句用途：

- 训练集：用于更新模型参数。
- 验证集：用于逐轮评估和选择最佳权重。
- 测试集：训练完成后独立复核。

仅展示接口真实返回的 `split_counts`，不推测背景图或实例数量。

### 关键配置

使用紧凑定义列表而非大面积带边框表格，显示：

- 基础模型、任务类型、训练设备。
- Batch、输入尺寸、优化器。
- 数据增强方案、提前停止、关闭 Mosaic。
- 数据集版本。
- 训练类别。

类别以可换行标签展示。关闭 Mosaic 文案明确为“最后 N 轮停止 Mosaic”，`0` 或缺失值按现有配置语义展示，不把它误写成全程关闭数据增强。

## 运行日志设计

新增 `TrainingLogsTab`，直接使用接口返回的最近 200 行日志。

工具栏提供：

- 模式切换：“实时日志”和“故障诊断”。故障诊断只显示包含常见错误或警告关键词的行。
- 搜索框：按文本即时过滤，大小写不敏感。
- 级别筛选：全部、轮次、警告、错误。
- 自动跟随开关：日志刷新后保持在底部；用户主动向上滚动后可以暂停跟随。
- 换行开关。
- 跳到底部按钮。
- 复制当前筛选结果按钮。
- 当存在 `runner_log` 产物时，显示“下载完整日志”。

日志行显示稳定行号，并根据内容分类着色：错误红色、警告黄色、轮次/进度青色、普通信息保持中性。空状态说明“训练启动并输出日志后显示”。

页签顶部固定提示：“页面展示最近 200 行；排查完整过程请下载完整日志”。筛选无结果时显示明确空状态，不隐藏工具栏。

日志视口桌面端高度约 `55vh`，移动端约 `50vh`；使用内部滚动，不随日志增长撑高整个抽屉。

## 组件与数据边界

新增或调整以下职责：

- `trainingDashboardPresentation.ts`：纯函数，负责最新指标选择、Loss 卡片生成、日志行分类和过滤。
- `TrainingOverviewTab.tsx`：仅负责概览布局和已有诊断组件组合。
- `TrainingLogsTab.tsx`：负责日志交互状态、滚动与复制/下载操作。
- `TrainingArtifactsTab.tsx`：只负责参数、权重和产物列表，不再渲染日志正文。
- `TrainingPage.tsx`：注册新的“运行日志”页签。
- `styles.css`：提供稳定尺寸、响应式网格和日志视口样式。

所有展示函数对缺失字段、空历史和历史版本运行保持兼容。

## 交互与错误处理

- 浏览器不支持剪贴板或复制失败时，使用现有 Ant Design 消息反馈，不影响日志查看。
- 没有完整日志产物时不显示下载按钮，而不是显示失效操作。
- 搜索与筛选只作用于当前接口返回的 200 行，界面明确该限制。
- 运行详情刷新时保留当前页签内的搜索、筛选、换行和跟随偏好；切换到另一条训练记录时清空搜索并回到默认筛选，避免把旧条件误用于新运行。
- 所有按钮使用现有 Lucide 图标，并提供可理解的文字或 tooltip。

## 响应式与视觉规范

- 桌面端以紧凑网格展示，核心指标与 Loss 分解最多四列。
- 中等宽度自动降为两列，移动端为单列。
- 使用现有中性色、蓝色和青绿色语义，不引入单一大面积主题色。
- 卡片圆角不超过 8px，不嵌套装饰性卡片。
- 数值字号与训练详情抽屉尺度匹配，不使用页面级大标题。
- 长模型名、数据集版本和类别标签允许换行或省略并提供 title，不能挤出容器。

## 测试与验收

### 单元测试

- 最新指标优先读取最新有效 epoch，空值不变成 0。
- 分割与检测任务生成正确的指标和 Loss 卡片。
- 最近 5 轮 Loss 趋势兼容缺失字段和 0 值。
- 日志分类识别错误、警告、轮次和普通行。
- 搜索、模式和级别筛选可组合，行号保持原始位置。
- 找到 `runner_log` 时生成完整日志下载入口。

### 组件测试

- 实时概览展示运行状态、真实指标、Loss 分解、数据划分用途和类别标签。
- 缺失验证指标时展示“首轮验证完成后生成”。
- 运行日志支持搜索、筛选、换行、复制和跳到底部。
- 参数与产物不再重复显示日志正文，但仍列出完整日志产物。

### 回归验证

- 运行相关前端测试套件。
- 运行完整前端测试。
- 执行 TypeScript / Vite 生产构建。
- 在当前本地地址分别检查桌面和移动视口，确认抽屉内容不溢出、日志区域可独立滚动。
- 验证过程中不停止或重启 API，不调用训练取消接口。

## 完成标准

- 用户无需打开曲线页即可理解当前阶段、时间、主要质量指标和各项 Loss。
- 缺失值说明何时产生，不再使用含糊的“待生成”。
- 日志可从独立页签直接进入，并能在最近 200 行内搜索和定位错误。
- 完整日志仍可下载，参数与产物页不再冗长。
- 桌面和移动端均无文本重叠、横向溢出或日志撑高页面的问题。
- 前端测试与生产构建通过，当前训练进程未被干预。
